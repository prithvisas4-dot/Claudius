Master the

Context Window

Claude works within a limited attention space. Everything it reads, analyzes, or executes takes up that space. Misuse it and you lose precision. Master it and you unlock power.

  • 200K tokens — that's roughly 150,000 words Claude can hold at once
  • Critical info goes at the START and END — not the middle
  • /compact — use this command to free space mid-session
  • Mix too many tasks + loose files = responses lose precision

✓ Do This
Put critical instructions at the very start AND end of your prompt. Claude weighs both ends most heavily.
✗ Avoid This
Dumping 20 files + 5 tasks into one session. Precision collapses. Split into focused sessions instead.
✓ Pro Move
Use XML tags to structure large inputs. <task>, <context>, <rules> — Claude reads structure, not just text.
✓ Power Move
Feed Claude its own prior output as context. Summarize the last session, paste it in, continue with full memory.

Hooks

Step 10

Stay Safe with

Checkpoints

Checkpoints are safety snapshots before major changes. They record the exact state of your project so you can roll back instantly if something goes off track. Bold execution requires a safety net.

  • Create a checkpoint before any major refactor or risky change
  • Snapshots include all tracked files — 2.3MB stored, instant restore
  • Name them clearly: before-auth-rewrite, before-db-migration
  • Restore with one command — no git stash juggling required
The Checkpoint Lifecycle

Create checkpoint before the big change
claude checkpoint create before-auth-rewrite
2
Claude makes all the changes

3

4

CLAUDE.md is the file that teaches Claude your entire project before it writes a single line. It's your operator manual, your style guide, your rules engine — all in one markdown file that lives in your project root.

Anatomy of a Perfect CLAUDE.md
# Project
Name, what the project is, one-line description. Claude reads this first — make it crisp.
## Stack
Languages, frameworks, libraries in use. What Claude should and should NOT introduce.
## Rules
Non-negotiable rules. "Always commit after changes." "Never use class components." "Use Phosphor icons only."
## Patterns
Code patterns to follow. File naming conventions. Folder structure. How components should be shaped.
## Context
Current state of the project. What's built. What's next. What's broken. Update this after every session.

What are Claude artifacts?

Artifacts are standalone, interactive outputs that Claude creates in a dedicated side panel — separate from the chat. Instead of plain text responses, you get live, working creations you can see, use, and share instantly.

Launched
2024 with Claude 3.5
Plans
All plans (free & paid)
No Coding
Just describe what you want
Think of it this way: instead of Claude writing code you copy somewhere else, the artifact IS the working app or document — right there in your browser.

When does Claude create an artifact?

Claude automatically creates an artifact when the output meets these criteria:

1
Substantial & self-contained
Typically 15+ lines long, something that stands on its own without needing conversation context.
2
Likely to be reused or iterated on
Content you'll want to edit, build upon, or reference again in the future.
3
Benefits from a live preview
Code, UIs, documents, and visuals that make more sense rendered than as raw text.
Short responses stay in chat. A quick paragraph or a few sentences will appear inline — artifacts are reserved for meaningful standalone outputs.

Types of artifacts you can create

Artifacts come in several formats, each rendered differently in the side panel:

React components (.jsx)
Interactive UIs, dashboards, tools, games — anything with buttons, inputs, charts
HTML pages (.html)
Full web pages with embedded CSS & JS — great for landing pages and layouts
Markdown (.md)
Formatted documents, reports, guides, blog posts, essays
SVG graphics (.svg)
Vector diagrams, illustrations, icons, flowcharts
Mermaid diagrams (.mermaid)
Flowcharts, sequence diagrams, entity-relationship charts
Files (.docx, .xlsx, .pptx, .pdf)
Downloadable Office-style documents and presentations

How to enable artifacts

Artifacts is on by default for most users, but here's how to check and configure it:

1
Open Settings
Click your profile icon in the top-right corner of Claude.ai, then select "Settings".
2
Go to "Feature Preview" or "Capabilities"
Find the Artifacts toggle and make sure it's turned on.
3
Enable AI-powered artifacts (optional)
Also toggle on "AI-powered Artifacts" to let Claude embed its own API calls inside your apps.
4
Enable Code Execution (optional)
Turn on "Code execution and file creation" for faster inline edits and Office-format file generation.
Team & Enterprise admins can manage artifact settings at the org level, including MCP access.

How to write great artifact prompts

The quality of your prompt directly determines the quality of the artifact. Here's what works:

Be specific about what it should do
Instead of "make a tracker", say "a daily water intake tracker with an input field, a Log button, and a weekly progress chart below"
Describe the layout
Mention where elements should go — "buttons side by side", "chart below the form", "sidebar on the left"
Specify the format
Say "React component", "plain HTML page", "Markdown report", or "SVG diagram" if you have a preference
Avoid vague requests
"Make a study app" → generic. "Flashcard app with question/answer fields, flip button, and a card list" → much better
Tip: You can always iterate. Start with a rough version and refine it with follow-up messages rather than trying to get everything perfect in one prompt.

Hack 04

They Use
Git Worktrees

Run multiple Claude Code agents at the same time, working on the same things, without them breaking each other's work.

  • Agent 1 fixes bugs on branch A
  • Agent 2 builds features on branch B
  • Both share the same repo — zero conflicts
  • 1M context with Claude Enterprise
